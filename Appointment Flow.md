### **1\. Prerequisites & Schedule Visualization**

Before booking, the Front Desk user accesses the calendar interface.

* **View Options:** The user views provider schedules in Day, Week, or Month modes to identify available and booked slots.

* **Resource Management:** The system displays real-time calendar availability to prevent conflicts.

### **2\. Initiating the Booking (The "Happy Path")**

* Step 1: Select Time Slot  
  The user clicks directly on an available time slot on the calendar.  
* Step 2: Navigation to appointment form  
  User should be navigated to create appointment form with the selected slot datetime, provider and other information.

* Step 3: Define Appointment Details  
  User fill the remaining detials in appointmnet form.

* **Step 4: Automatic Validation (Conflict & Insurance)**  

If no conflicts, the system fetches the patient's insurance status and displays a color-coded icon indicating eligibility (e.g., verified, pending, failed) directly on the appointment form as well as list view.

* Step 5: Confirmation  
  The user confirms the details. The system creates the appointment with a status of "scheduled" and immediately displays it on the calendar and appointment list.
